Posted on 2021-01-11 Edited on 2021-02-28 In Microservices

项目初始化

新建虚拟环境

这里选用 Python ，主要用于最底层的各个微服务的 Service 层的开发

提前先创建了一个 Envs 的目录专门用来存放虚拟环境文件，进入 Envs, 创建一个目录 cat-srvs，然后进入该目录

此时虚拟环境就已经启动

新建 Python 项目

cat-srvs 该目录中存放的是所有的微服务，每个微服务新建一个 package 或者一个 文件夹，当然也可以对每个微服务新建一个项目，这里为了方便管理，所以每个微服务使用一个 package

用户服务

git branch: t1_project_init

第一个要完成的是 用户服务，首先在 cat-srvs 目录下新建 user_srv,这里每个 package 就是一个项目，user_srv 就是一个项目，然后在 user_srv 目录中新建一些文件夹来组织代码

结构如下：

  • handler: 用于存放所有的业务逻辑代码；
  • logs: 存放日志文件;
  • models: 存放 peewee 的 model 文件;
  • proto: 存放 proto 文件；
  • settings: 存放配置文件，类似其他框架中的 config 目录；

  • peewee 创建数据表(创建数据表示还有一些限制，比如设置空字符串怎么设置需要另外研究)

安装依赖包

安装 peewee 包 和 pymysql 包

数据库配置

首先创建数据库，会为每个微服务创建独立的数据库，所以这里创建一个 cat_user_srv 的数据库

在 settings 目录中添加 settings.py 文件

  • ReconnectMySQLDatabase 这个类主要是完成一个继承的过程
  • PooledMySQLDatabase 和 MySQLDatabase 基本差不多，但是使用 PooledMySQLDatabase 连接 MySQL 的时候如果一段时间不操作，这个连接会被关闭，关闭之后如果再使用 SELECT 查询就会抛异常 mysql gone away，所以还需要导入 ReconnectMixin,它的作用是当连接被关闭，当再次使用时会重新连接;
  • 使用时只需要定义一个类，直接继承 PooledMySQLDatabase 和 ReconnectMixin,类里面什么都不需要操作，主要是完成继承的过程; 这里可以去学习下 python 的 mro,集成属性的调用过程
