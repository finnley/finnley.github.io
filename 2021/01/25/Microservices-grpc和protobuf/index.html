Microservices-grpc和protobuf

Posted on 2021-01-25 Edited on 2021-02-17 In Microservices

grpc

gRPC 是一个高性能、开源和通用的 RPC 框架，面向移动和 HTTP/2 设计。目前提供 C、Java 和 Go 语言版本，分别是：grpc, grpc-java, grpc-go. 其中 C 版本支持 C, C++, Node.js, Python, Ruby, Objective-C, PHP 和 C# 支持

gRPC

比如在 Ruby 编写了一套 Client，Client 调用的时候通过 Stub 向服务端发送请求，先将数据转换成 protobuf 协议，通过 protobuf 工具进行序列化成二进制发送到服务端，然后服务端进行反序列化，类似json的 json.dumps() 和 json.loads() 的过程。

protobuf

java中的dubbo dubbo/rmi/hessian messagepack 如果你懂了协议完全有能力自己去实现一个协议

• 习惯用 Json、XML 数据存储格式的你们，Protocol Buffer 的缩写
• Protocol Buffer 其实 是 Google 出品的一种轻量 & 高效的结构化数据存储格式，性能比 Json、XML 真的强！太！多！
在考虑一种协议时，需要考虑数据压缩效率怎样，如果数据压缩效率还行，但是在序列化和反序列化时效率不高，即性能不高时任然会产生影响，Protocol Buffer 相对于 Json 和 XML 都要强上许多
• protobuf经历了protobuf2和protobuf3，pb3比pb2简化了很多，目前主流的版本是pb3

生成proto的python文件

  • --python_out=.: . 表示生成的 python 文件放在当前目录下
  • --grpc_python_out=.: 可以生成 grpc 的相关文件,生成的文件也在当前目录下
  • -I.: I 表示 Input,到哪个目录找哪个文件进行生成，. 就表示当前目录,如果不是在当前目录，. 就需要改成对应的目录
  • hello.proto: 到当前目录下查询 helloworld.proto 文件，生成 protobuf 和 grpc要用到的文件，并生成到当前目录下
  • 整体的意思就是到当前目录下查找 hello.proto 文件，分别生成两个 python 文件，第一个文件时 protobuf 需要用到的，第二个是 grpc 需要用到的，两个文件都是生成到当前目录之下,当执行完之后，当前目录路会多出两个文件 hello_pb2.py 和 hello_pb2_grpc.py
