代码逻辑

** 首先判断是否是 GET 请求还是 POST 请求
** 如果是 GET 提请求就直接展示登录页面
** 如果是 POST 请求则进行代码逻辑处理
** 获取登录用户名 login_name 和 登录密码 login_pwd,并将他们保存到变量中
** 对上一步中获取的参数进行基本的验证（只要是用书输入的信息我们都不要去相信它）
例如：如果没有输入用户名和密码则提示输入正确的用户名和密码
** 业务逻辑判断
例如：从用户表中查找登录用户名这个字段等于输入的登录用户名这条记录是否存在，如果存在则进行密码验证，如果不存在就提示他输入正确的用户名和密码。

密码验证
密码加密算法(数据库中保存的密码) = MD5(当前用户输入的密码 + MD5(数据库中登录密码随机密钥的)) = md5(login_pwd + md5(login_salt))

如果加密后的密码不等于数据库中保存的密码就证明密码错误，则提示输入正确的用户名和密码

** 保存用户登录状态（这使用Cookie进行保存）
登录成功后需要将用户的状态保存下来，后面就可以根据登录态知道是哪个用户登录的。Cookie的值是有一个结构的:
Cookie Value = 加密字符串 + “#” + uid 加密字符串 = md5(login_name + login_pwd + login_salt)

** 最后进行页面跳转，这里默认的跳转到仪表盘页面

统一登录验证

实现到这里，其实我们做系统有会有一个共识，就是后台管理系统所有页面都是必须登录之后才可以访问，而现在的所有页面是在没有登录的情况下都可以进行访问，所以这里需要做统一登录验证。

做统一登录验证，我们不可能在每一个方法之前去写统一登录验证，或者是在析构方法里面去验证，这样做实际上是麻烦的，这里需要引用一个统一拦截器的概念，让可以在每一个方法执行之前执行一个制定的方法，这样我们可以在制定的方法里面统一验证。

我们在 modules/web/controller 控制器下创建一个名为 common 的文件夹，在里面创建一个名为 BaseController 的控制器，然后这个控制器还需要继承统一控制器 BaseWebController ,该控制的作用是做一些web独有的验证，比如指定特定的布局文件和进行登录验证

![Book Shop](/images/book-shop/09/4.png)

代码逻辑

** 首先定义一个私有方法用户检查登录状态，要么返回false，要么返回true
获取cookie，将cookie存放到一个变量中；
** 判断 尽量列出所有可能的结果

  1. 如果cookie不存在，则返回false;
  2. 如果cookie存在，将字符串以 # 分割，分别存入两个不同的变量中一个是加密字符串一个是用户编号 uid ;
  3. 判断字符串或者uid不存在证明cookie被篡改过，返回false;
  4. 对 uid 进行有效性验证,比如如果 uid 不是数字则返回false;
  5. 根据 uid 到用户表中查找该 uid 的记录并存到一个变量中，判断该条记录是否存在，若不存在则返回false;
  6. 验证加密字符串的有效性。将上一步查询出来的登录名和登录密码和登录随机密钥拼接进行md5加密并保存到变量中与步骤2中的字符串进行比较，如果不同则返回false;
  7. 其他情况返回true
    ** 然乎在拦截器代码逻辑，调用刚才定义的私有方法并将值保留到变量中，如果没有登录的情况下，一种是ajax请求，一种是其他请求，如果是ajax请求则提示ajax信息，提示登录信息；其他情况则跳转到登录页面。
