Posted on 2019-09-28 Edited on 2020-07-01 In Logstash

Filter Plugin

Filter 是 Logstash 功能强大的主要原因，他可以对 Logstash Event 进行丰富的处理，比如解析数据，删除字段，类型转换等等，常见的有如下几个：

  • date 日期解析
  • grok 正则匹配解析
  • dissect 分割符解析
  • mutate 对字段作处理，比如重命名，删除，替换等
  • json 按照 json 解析字段内容到指定字段中
  • geoip 增加地理位置数据
  • ruby 利用 ruby 代码来动态修改 Logstash Event

date 参数

  • match

    • 类型为数组，用于指定日期匹配的格式，可以一次指定多种日期格式
    • match => [“logdate”, “MMM dd yyyy HH:mm:ss”, “MMM d yyyy HH:mm:ss”, “ISO8601”]

  • target

    • 类型为字符串，用于指定赋值的字段名，默认是 @timestamp

  • timezone

    • 类型为字符串，用于指定时区

grok

Grok 语法：

  • %{SYNTAX:SEMANTIC}

  • SYNTAX 为 grok pattern 的名称，SEMantic 为赋值字段名称

  • %{NUMBER:duration} 可以匹配数值类型，但是 grok 匹配出的内容都是字符串类型，可以通过在最后指定为 int 或者 float 来强制转换类型

%{NUMBER:duraton:float}

dissect

基于分割符原理解析数据，解决grok解析时消耗过多 cpu 资源的问题

对比 grok 解析日志会， dissect 会更加的简洁

dissect 的应用有一定的局限性

* 主要适用于每行格式相似且分隔符明确简单的场景

dissect 语法比较简单，有一些列字段 (field) 和分隔符 (delimiter) 组成

* %{} 字段

* %{} 之间是分隔符

例子：

  • dissect 语法

hi - hello - 12

%{a} - %{b} - %{c}

如果不写名称，即 %{} ,表明忽略该值

“a” 为字段名
“hi”: 为匹配值

Apr 26 12:20:02

%ts} %{+ts} %{+ts}

  • 代表该匹配的值追加到 ts 字段下

two three one go

%{+order/2} %{+order/3} %{+order/1} %{+order/4}

/ 后面的数组代表拼接的次序

a=1&b=2

%{?key1}=%{&key1}&%{?key2}=%{&key2}

%{?} 代表忽略匹配值，但是赋予字段名。用于后续匹配用
%{&} 表示将匹配值赋予 key1 的匹配值

{
“a”: 1,
“b”: 2
}

dissect 可以自动处理空的匹配值

Join Smith,Big Oaks,Wood Lane,Hmbledown,Canterbury,CB34RY

%{name},%{addr1},%{addr2},%{addr3},%{city},%{zip}

Jane Doe,4321 Fifth Avenu,,,New York,87432

dissect 分割后的字段值都是字符串，可以使用 convert_datatype 属性进行类型转换

mutate

使用最频繁的操作，可以对字段进行各种操作，比如重命名，删除，替换，更新等，主要操作如下：

* convert 类型转换

* gsub 字符串替换

* split/join/merge 字符串切割，数组合并字符串，数组合并为数组

* rename 字段重命名

* update/replace 字段内容更新或替换

* remove_field 删除字段
  1. convert

output

负责将 Logstash Event 输出，常见的插件如下

  • stdout

  • file

  • elasticsearch

stdout

输出到标准输出，多用于调试

file

输出到文件，实现将分散在多地的文件统一到一处的需求，比如讲所有 web 机器的web 日志手机到1个文件中，从而方便查阅信息
