docker介绍

Posted on 2019-05-08 Edited on 2020-05-24 In Docker

Docker能做什么？

  • 简化配置

可以实现将源代码， 运行环境以及配置都打包到一个容器里面，并且这个容器可以运行在不同的环境中，因为简化所以自然而然的就提高了开发效率。

  • 整合服务器

  • 代码流水线管理

  • 调试能力

  • 提高开发效率

可以实现本地开发环境，测试环境和生产环境非常接近， 这样可以极大的减少在开发和测试以及部署过程中因为环境的不同而产生的错误。

  • 多租户

  • 隔离应用

  • 快速部署

容器技术概述

传统部署应用流程：

如果要部署一个APP，需要准备一台物理服务器，然后在这个物理服务器上安装系统，这个操作系统可能是Windows,也可能是Linux，然后在这个操作系统里面去部署Application。

  • 缺点

  1. 部署非常慢

    比如需要购买服务器，布置机房，安装操作系统，然后需要安装App，安装App的时候可能还需要系统环境的依赖

  2. 成本高

    需要购买物理服务器

  3. 资源浪费

    可能服务器资源太多了，如内存根本就用不完

  4. 难以迁移和扩展

    如果迁移，可能需要重新购买服务器，布置机房等

  5. 可能会被限定硬件厂商

虚拟化技术

  • 虚拟化技术实现方式

    在原先物理服务器上面通过 Hypervisor 去做物理资源的虚拟机化，比如将CPU资源，内存资源，硬盘资源进行虚拟化，然后在这些虚拟化之上再去安装操作系统，这些操作系统其实就是虚拟机，如此一来， 一个物理机就可以部署多个App ,并且 每个App独立运行在一个VM里面

  • 优点

    1. 资源池

      一个物理机的资源可以分类到不同的虚拟机里面，然后去做物理资源的管理

    2. 易扩展

      增加物理机器或者增加虚拟机

    3. 易于云化

      亚马逊AWS、阿里云、腾讯云等

  • 缺点

    1. 每个虚拟机都是一个完整的操作系统，要给其分配资源，当虚拟机数量增多时，操作系统本身消耗的资源势必增多

容器

  • 解决了开发和运维之间的矛盾

  • 在开发和运维之间搭建了一个桥梁，是实现devops的最佳解决方案

什么是容器？

  • 容器是对软件和其依赖的标准化打包

  • 容器可以实现应用之间的相互隔离

  • 容器共享同一个OS Kernel

  • 容器可以运行在很多主流操作系统上
