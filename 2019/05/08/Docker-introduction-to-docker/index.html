Docker介绍

Posted on 2019-05-08 Edited on 2020-11-29 In Docker

Docker能做什么？

  • 简化配置
    可以实现将源代码， 运行环境以及配置都打包到一个容器里面，并且这个容器可以运行在不同的环境中，因为简化所以自然而然的就提高了开发效率。
  • 整合服务器
  • 代码流水线管理
  • 调试能力
  • 提高开发效率
    可以实现本地开发环境，测试环境和生产环境非常接近， 可以极大的减少在开发和测试以及部署过程中因为环境的不同而产生的错误。
  • 多租户
  • 隔离应用
  • 快速部署

容器技术概述

传统部署应用流程：

如果要部署一个 APP，需要准备一台物理服务器，然后在这个物理服务器上安装系统，这个操作系统可能是 Windows ,也可能是 Linux，然后在这个操作系统里面去部署 Application。

  • 缺点

  1. 部署非常慢

    比如需要购买服务器，布置机房，安装操作系统，然后需要安装App，安装App的时候可能还需要系统环境的依赖

  2. 成本高

    需要购买物理服务器

  3. 资源浪费

    可能服务器资源太多了，如内存根本就用不完

  4. 难以迁移和扩展

    如果迁移，可能需要重新购买服务器，布置机房等

  5. 可能会被限定硬件厂商

虚拟化技术

  • 虚拟化技术实现方式

    在原先物理服务器上面通过 Hypervisor 去做物理资源的虚拟机化，比如将CPU资源，内存资源，硬盘资源进行虚拟化，然后在这些虚拟化之上再去安装操作系统，这些操作系统其实就是虚拟机，如此一来， 一个物理机就可以部署多个App ,并且 每个App独立运行在一个VM里面

  • 优点

    1. 资源池

      一个物理机的资源可以分类到不同的虚拟机里面，然后去做物理资源的管理

    2. 易扩展

      增加物理机器或者增加虚拟机

    3. 易于云化

      亚马逊AWS、阿里云、腾讯云等

  • 缺点

    1. 每个虚拟机都是一个完整的操作系统，要给其分配资源，当虚拟机数量增多时，操作系统本身消耗的资源势必增多

容器

  • 解决了开发和运维之间的矛盾

  • 在开发和运维之间搭建了一个桥梁，是实现devops的最佳解决方案

什么是容器？

  • 容器是对软件和其依赖的标准化打包

  • 容器可以实现应用之间的相互隔离

  • 容器共享同一个OS Kernel

  • 容器可以运行在很多主流操作系统上

容器和虚拟机的区别

为什么使用Docker虚拟机

我们可以在Linux系统上直接安装程序，然后部署项目，那为什么还要安装docker虚拟机，然后在docker虚拟机内部去安装程序部署项目呢？

主要是为了解决隔离性的问题， 有些程序运行会占用大量的硬件资源，比如芒果数据库，运行的时候会占用大量的内存和创建缓存，这点势必会影响到其他程序的运行；

再有使用虚拟机去部署程序会非常的方便，比如我需要部署程序，只需要开启一个虚拟的空间，然后我不需要部署程序了，我只要把虚拟空间删除掉就行了，这些虚拟空间是完全隔离的。

如果不使用虚拟机，当在Linux系统内安装程序，我要把A程序卸载掉，这个程序关联了一些软件包，当我把A程序卸载掉可能会影响到B程序，B程序跑不起来了，那么这个就体现不了隔离性。这一点Docker虚拟机是可以完全体现隔离性的。把A容器删除掉之后，这个A容器就是Docker虚拟机里面的虚拟空间，我把A容器删掉不会影响到B容器，这个隔离性就非常的好，这就是使用Docker虚拟机去部署项目的主要原因。

Docker虚拟机是一个轻量级的虚拟机，VM虚拟机是一个重量的虚拟机。我们在一个服务器主机上开启Docker虚拟机之后，创建几千个虚拟的空间，然后一起去启动，一点问题都没有。

Docker虚拟机架构

Docker创建的所有虚拟实例共占用同一个Linux内核，对硬件占用较小，属于轻量级虚拟机。

我们在硬件环境之上安装了一个操作系统，这个就是宿主机系统，比如我们安装的 CentOS 系统。

然后我们在Linux系统内上去安装Docker程序，这个Docker程序之内含有的Docker引擎会去管理，如创建，销毁，停止这些虚拟空间，在这些虚拟空间之内我们可以去安装程序，部署项目，这些虚拟空间我们叫做容器。容器和容器是完全隔离的。

需要部署什么东西，我们只需要开启一个容器，然后在里面安装程序，部署项目，不需要可以直接把容器删除掉就行了，不会对其他容器造成影响。

Docker为什么被叫做轻量级的虚拟机呢？

原因是因为这些容器里面运行的linux系统并不是独立的完整的linux,这些容器运行的linux其实是共用了一个linxu内核，就是Docker软件为它们提供的linux系统内核。

而VM虚拟机不是这样的，VM虚拟机每一个虚拟的空间都是运行的一个独立的系统。

正因为很多容器是共享一个linux内核，也就是说我们在Docker虚拟机内是没办法安装其他的操作系统的，只能使用Docker提供的Linxu系统，然后在容器里面去部署基于linux系统的程序。

Docker镜像与容器

镜像是用来创建容器的；
容器是从镜像中创建出来的实例

镜像的内容是只能读取，不能写入，而容器是可读可写的。这就意味着我们部署项目是部署在容器里面，而不是部署在镜像里面，镜像不是虚拟的空间，容器才是虚拟的空间。创建容器的时候，我们可以指定这个虚拟的空间多大的内存，几核的CPU，什么样的网络环境，所以容器是可以运行的，而镜像是不能运行的，镜像是一个文件，它里面安装了你想要的一些程序。

既然镜像是只读，那我们怎们往镜像里安装程序呢？

其实Docker还有一种文件叫 DockerFile，我们在编写 Dockerfile 的时候可以定义我们想要安装的程序，写好这些程序之后，Dockerfile 运行时就会把这些程序安装上，然后创建出镜像。

当然我们也可以在容器里面安装一些程序，然后把容器转换成镜像。
